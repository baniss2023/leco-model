Les structures conditionnelles

Les structures conditionnelles sont essentielles en programmation, car elles permettent à un programme de prendre une décision. Sans elles, un programme exécuterait toujours les mêmes instructions dans le même ordre. Grâce aux conditions, il peut adapter son comportement selon une situation précise.

Comprendre l’idée d’une condition

Qu’est-ce qu’une condition ?

Une condition est une expression qui permet de répondre à une question par True ou False. En d’autres mots, une condition sert à tester si quelque chose est vrai ou faux.

Exemples de conditions

x > 0
note >= 60
age == 18
nom != "Ali"
Idée clé : une condition ne produit pas une phrase complète ; elle produit une réponse logique : vrai ou faux.

Pourquoi utilise-t-on des conditions ?

On utilise les conditions lorsqu’on veut que le programme fasse un choix. Au lieu d’exécuter toujours la même suite d’instructions, le programme adapte son comportement à la situation rencontrée.

  • Si la note est suffisante, afficher « réussite » ; sinon, afficher « échec ».
  • Si le mot de passe est correct, autoriser l’accès ; sinon, refuser l’accès.
  • Afficher un message différent selon la valeur entrée par l’utilisateur.
À retenir : une structure conditionnelle permet de contrôler le flux d’exécution d’un programme.

Expressions, comparaisons et booléens

Expression arithmétique et expression booléenne

Une expression arithmétique produit une valeur numérique. Une expression booléenne produit quant à elle True ou False. Cette distinction est importante, car une structure conditionnelle teste toujours une expression booléenne.

Expression arithmétique

5 + 2
x * 3
a - b

Expression booléenne

x < 10
note >= 60
age == 18

Les opérateurs de comparaison

Les opérateurs de comparaison servent à fabriquer des tests. Ils permettent de comparer deux valeurs pour savoir si une condition est vraie ou fausse.

OpérateurSignification
<inférieur à
<=inférieur ou égal à
>supérieur à
>=supérieur ou égal à
==égal à
!=différent de
  • x < 5 → est-ce que x est plus petit que 5 ?
  • note >= 60 → est-ce que la note est au moins 60 ?
  • mot != "admin" → est-ce que le mot est différent de admin ?

Attention : affectation et comparaison

Affectation

L’affectation sert à donner une valeur à une variable.

Exemple

x = 3

Ici, on place la valeur 3 dans la variable x.

Comparaison

La comparaison sert à vérifier si deux valeurs sont égales.

Exemple

x == 3

Ici, on demande : « est-ce que x vaut 3 ? »

À retenir absolument : = sert à affecter une valeur, alors que == sert à comparer deux valeurs.

Exemple d’erreur classique

C’est une erreur fréquente en programmation. En Python, on ne peut pas écrire une affectation à la place d’un test dans un if.

condition_incorrecte.py
Version incorrecte
1if x = 3:
2print("oui")
condition_correcte.py
Version correcte
1if x == 3:
2print("oui")

Conditions composées

Les opérateurs logiques

Parfois, une seule condition ne suffit pas. On doit alors combiner plusieurs tests. En Python, on utilise principalement and, or et not.

  • and : et
  • or : ou
  • not : non

Exemples de conditions composées

Exemples

x > 0 and x <= 10
x < 0 or x > 100
not x == 0
x > 0 and x <= 1
x != 0
  • x > 0 and x <= 10 signifie que x est strictement supérieur à 0 et inférieur ou égal à 10.
  • x < 0 or x > 100 signifie que x est inférieur à 0 ou supérieur à 100.
  • not x == 0 signifie que x n’est pas égal à 0.
  • x > 0 and x <= 1 signifie que x appartient à l’intervalle ]0, 1].
  • x != 0 signifie que x est différent de zéro.
Conseil pédagogique : quand une condition devient longue, il faut la lire lentement en français avant de la coder.

Structure conditionnelle réduite

Définition

La structure conditionnelle réduite permet d’exécuter un traitement seulement si une condition est vraie. Si la condition est fausse, le programme ne fait rien dans ce bloc.

Forme en algorithmique et en Python

Cette structure correspond à la forme la plus simple d’un test conditionnel.

algorithme.algo
Pseudo-code — logique algorithmique
1Si condition Alors
2traitement
3Fin Si
structure_reduite.py
Fichier Python — structure générale
1if condition:
2traitement
  • if signifie si ;
  • le caractère : annonce le début du bloc ;
  • le traitement doit être indenté.

Exemple : tester la divisibilité par 11

Dans cet exemple, on exécute un traitement seulement si le nombre saisi est divisible par 11.

divisible_par_11.py
Fichier Python — exemple
1x = int(input("Entrez un entier : "))
2if x % 11 == 0:
3print(f"{x} est divisible par 11")
Exemple possible avec x = 22
22 est divisible par 11
  • input() lit la saisie clavier ;
  • int() transforme la saisie en entier ;
  • x % 11 donne le reste de la division de x par 11 ;
  • si ce reste vaut 0, alors x est divisible par 11.

Autre exemple simple

Structure conditionnelle avec alternative

Définition

La structure conditionnelle avec alternative permet de choisir entre deux traitements. Si la condition est vraie, on exécute le premier bloc ; sinon, on exécute le second.

Forme en algorithmique et en Python

algorithme.algo
Pseudo-code — logique algorithmique
1Si condition Alors
2traitement 1
3Sinon
4traitement 2
5Fin Si

2traitement_1

4traitement_2

4else:

Exemple possible avec n = 7
Nombre impair
  • si le reste de la division par 2 vaut 0, le nombre est pair ;
  • sinon, il est impair.

Point important : dans une structure if / else, un seul bloc est exécuté.

Structure conditionnelle généralisée

Définition

La structure conditionnelle généralisée permet de gérer plusieurs cas. Le programme teste les conditions dans l’ordre. Dès qu’une condition est vraie, il exécute le bloc correspondant et quitte la structure.

Forme en algorithmique et en Python

algorithme.algo
Pseudo-code — logique algorithmique
1Si condition 1 Alors
2traitement 1
3Sinon Si condition 2 Alors
4traitement 2
5Sinon Si condition 3 Alors
6traitement 3
7Sinon
8traitement final
9Fin Si
if_elif_else.py
Fichier Python — structure générale
1if condition1:
2traitement1
3elif condition2:
4traitement2
5elif condition3:
6traitement3
7else:
8traitement_final

Pourquoi l’ordre des conditions est très important

Quand plusieurs conditions se ressemblent, leur ordre peut changer complètement le résultat.

ordre_correct.py
Version correcte
1note = 85
2if note >= 90:
3print("A")
4elif note >= 80:
5print("B")
6elif note >= 70:
7print("C")
8else:
9print("D")
ordre_problematique.py
Version problématique
1note = 85
2if note >= 70:
3print("C")
4elif note >= 80:
5print("B")
6elif note >= 90:
7print("A")
Règle à retenir : quand plusieurs conditions se ressemblent, il faut souvent commencer par la plus précise ou la plus restrictive.

Structure conditionnelle à choix multiple

Définition

La structure à choix multiple sert à sélectionner un traitement parmi plusieurs possibilités qui dépendent toutes de la même variable. Dans d’autres langages, on parle parfois de switch ou de case.

Forme en algorithmique et représentation en Python

choix_multiple.algo
Pseudo-code — logique algorithmique
1Selon sélecteur Faire
2Valeur 1 : Traitement 1
3Valeur 2 : Traitement 2
4Valeur 3 : Traitement 3
5Sinon : Traitement final
6Fin Selon

En Python

Dans cette leçon, on représente généralement ce type de décision avec une suite de if, elif et else.

Exemple

1jour = int(input("Entrez un numéro de jour : "))
2if jour == 1:
3print("Lundi")
4elif jour == 2:
5print("Mardi")
6elif jour == 3:
7print("Mercredi")
8elif jour == 4:
9print("Jeudi")
10elif jour == 5:
11print("Vendredi")
12else:
13print("Valeur invalide ou autre cas")
Exemples possibles
jour = 1  -> Lundi
jour = 4  -> Jeudi
jour = 9  -> Valeur invalide ou autre cas

Le sélecteur est souvent une seule variable : un entier, un caractère ou un booléen. On compare cette même variable à plusieurs valeurs possibles.

Mal ordonner les conditions et compliquer inutilement un test

Dans une structure if / elif / else, l’ordre change le résultat. Il faut aussi éviter d’écrire une condition inutilement compliquée quand une forme plus simple existe.

condition_longue.py
Écriture plus lourde
1if x < 0 or x > 0:
2print("x est différent de 0")
condition_simple.py
Écriture plus claire
1if x != 0:
2print("x est différent de 0")

À retenir absolument

Points à retenir

  • Une condition produit True ou False.
  • Les opérateurs de comparaison servent à construire des tests.
  • = ne veut pas dire la même chose que ==.
  • if sert à tester une condition.
  • else sert au cas contraire.
  • elif sert à ajouter d’autres cas.
  • L’indentation en Python est obligatoire.
  • L’ordre des conditions peut changer le résultat du programme.
Message pédagogique final : plus tu apprends à lire correctement une condition, plus tu deviens capable de construire des programmes fiables, clairs et utiles.
